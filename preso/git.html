Siempre se trabaja desde una rama, nunca se incorpora a main o master sin que se haya revisado el código, aunque sea por uno mismo. Es esencial trabajar en las ramas.

Los PRs siempre desde una rama

Resolviendo conflictos

git checkout --theirs fichero

Si todo falla

git clone url
git rm origin
git remote add origin mi-fork-de-url
git push --force
Cuidado con hacer esto con un PR abierto; si se hace así, se cerrará. En todo caso, esto no debería ocurrir si siempre se pide PR desde una rama.
